# 1. 프로젝트 개요

이 프로젝트는 스타벅스 코리아 웹사이트 메인 페이지를 참고하여 제작된 반응형 웹사이트입니다.

구성 기술:

- HTML5
- CSS3
- jQuery 3.7.1
- Swiper 11 (슬라이드 라이브러리)

이 사이트는 기업형 랜딩 페이지 구조를 학습하거나 포트폴리오용으로 사용하기에 적합한 구조로 설계되어 있습니다.

---

# 2. 전체 파일 구성 이해

## 2.1 HTML 구조

HTML은 크게 다음과 같은 구조로 나뉩니다.

```
<body>
 ├──overlay
 ├── fixedTop
 ├──header
 ├──section.hero
 ├──section.depth1
 ├──section.depth2
 ├──section.depth3
 ├──section.depth4
 ├──section.depth5
 └──footer
```

각 영역은 독립적인 콘텐츠 블록으로 구성되어 있으며, 유지보수와 확장이 용이하도록 분리되어 있습니다.

---

# 3. 각 영역별 상세 설명

## 3.1 header 영역

header는 사이트의 최상단 네비게이션을 담당합니다.

### 내부 구성

1. search-wrap
    
    검색창 레이어입니다.
    
    header에 schOpen 클래스가 추가되면 표시됩니다.
    
2. h-dep1
    
    상단 로고, SNS 아이콘, 유틸 메뉴(검색, 위치정보, 모바일 메뉴 버튼)
    
3. h-dep2
    
    메인 네비게이션 메뉴 (커피, 음료, 푸드 등)
    
4. sub-nav-wrap
    
    마우스 오버 또는 모바일 메뉴에서 열리는 2차 메뉴
    

### 특징

- 모바일에서는 h-dep2가 숨겨지고 sub-nav-wrap이 슬라이드 메뉴로 동작합니다.
- body에 mob-mOpen 클래스가 추가되면 모바일 메뉴가 열립니다.

---

## 3.2 hero 섹션

메인 비주얼 슬라이드 영역입니다.

사용 라이브러리: Swiper 11

구조:

```
.swiper.hero-slider
  ├── .swiper-wrapper
  │     ├── .swiper-slide
  │     └── .swiper-slide
  └── .swiper-pagination
```

각 슬라이드는 배경 이미지를 CSS로 적용합니다.

이 영역은 사이트의 브랜드 이미지를 강조하는 핵심 영역입니다.

---

## 3.3 depth1 (뉴스 + 이용안내)

좌측: 새소식 슬라이드

우측: 이용안내 / 이용현황 버튼

news-slider 역시 Swiper로 구현되어 있으며,

이전/다음 버튼이 커스터마이징되어 있습니다.

---

## 3.4 depth2 (HOT MENU)

탭 기반 슬라이드 영역입니다.

구성:

- 상단: 탭 메뉴
- 하단: 4개의 개별 Swiper 슬라이드

```
.dep2-sl-1
.dep2-sl-2
.dep2-sl-3
.dep2-sl-4
```

탭 클릭 시 Active 클래스를 변경하여 해당 슬라이드만 표시됩니다.

이 구조는 메뉴 카테고리별 상품을 분리하기 위한 설계입니다.

---

## 3.5 depth3 (콘텐츠 섹션)

3개의 콘텐츠 블록으로 구성됩니다.

1. 신상 메뉴 (고정 레이아웃)
2. 스타벅스와 문화 (슬라이드)
3. 행사/전시 (슬라이드)

좌측은 고정 카드형 콘텐츠,

우측은 Swiper 기반 슬라이드 구조입니다.

---

## 3.6 depth4 (연도별 카드)

2025, 2024, 2023 카드 슬라이드

각 연도별로 개별 Swiper 인스턴스를 사용합니다.

구조 예시:

```
.card2025-slider
.card2024-slider
.card2023-slider
```

각 슬라이드는 이미지 + 텍스트 오버레이 구조입니다.

---

## 3.7 depth5 (공지사항 / 언론보도)

좌측: 탭 전환 구조

- 공지사항
- 언론보도

클래스:

- Active → 콘텐츠 표시
- on → 탭 활성화

우측:

- 개장안내
- 자주하는 질문

고정 배너형 UI 구조입니다.

---

## 3.8 footer

2단 구조입니다.

1단:

- 관련 사이트 슬라이드 (Swiper 사용)

2단:

- 회사 정보
- 개인정보 관련 링크
- 하단 배너

- Flexbox 기반 레이아웃

## 4.2 반응형 기준

브레이크포인트:

- 1260px 이하
- 1024px 이하
- 920px 이하

1024px 이하에서:

- header 고정
- 모바일 메뉴 활성화
- h-dep2 숨김
- sub-nav-wrap 슬라이드 방식 전환

# 6. 클래스 네이밍 규칙 이해

형식:

- depth1, depth2 → 섹션 단위 구분
- in-wrap → 내부 카드 블록
- rel → 가운데 정렬 wrapper
- Active → 표시 제어
- On → 상태 활성화

이 규칙을 이해하면 유지보수가 쉬워집니다.

# 7. 수정 시 참고 가이드

## 7.1 슬라이드 추가 방법

1. 해당 swiper-wrapper 내부에
    
    ```
    <divclass="swiper-slide">...</div>
    ```
    
    추가
    
2. JS에서 해당 슬라이드 초기화 확인

---

## 7.2 메뉴 추가 방법

- h-dep2의 main-nav에 li 추가
- sub-nav-wrap의 동일 위치에 서브메뉴 추가

---

## 7.3 콘텐츠 교체 방법

이미지 변경:

```
background-image:url(...)
```

또는

```
<imgsrc="...">
```

텍스트 변경:

해당 h2, h3, span 수정
